# Toon App AIエージェント向けガイドライン

このドキュメントは、`toon-app`プロジェクトに携わるAIエージェントのための包括的なガイドです。一貫性と品質を確保するために、プロジェクト構成、技術スタック、デザイン原則、コーディング規約について概説します。

## 1. プロジェクト概要
**プロジェクト名:** Toon App (Gemini Quiz App)
**説明:** クイズインターフェース、ダッシュボード、RAG (Retrieval-Augmented Generation) ボットを備えたWebアプリケーション。
**目標:** 魅力的で視覚的に美しく、インタラクティブなユーザー体験を提供すること。

## 2. 技術スタック
- **コア:** React (Vite), TypeScript
- **スタイリング:** Vanilla CSS (Modules または グローバル `index.css`)。
    - *注記:* ユーザーから明示的な要求がない限り、TailwindCSSの使用は避けてください。
- **ビルドツール:** Vite
- **パッケージマネージャー:** npm

## 3. デザインと美学
**主要原則:** 「視覚的卓越性 (Visual Excellence)」と「豊かな美学 (Rich Aesthetics)」
- **カラーパレット:** 鮮やかで調和のとれた色を使用してください（例：HSL調整された色）。一般的なブラウザのデフォルト色は避けてください。
- **タイポグラフィ:** モダンなフォント（例：Inter, Roboto, Outfit）を使用してください。
- **インタラクティビティ:**
    - スムーズなトランジションとホバー効果。
    - エンゲージメントを高めるためのマイクロアニメーション。
    - 適切な場所でのグラスモーフィズムや洗練されたダーク/ライトモード。
- **レスポンシブ:** デスクトップ、タブレット、モバイルでレイアウトがシームレスに動作するようにしてください。

## 4. ディレクトリ構成
```
/src
  /assets       # 静的アセット (画像, アイコン)
  /components   # 再利用可能なUIコンポーネント
  /content      # Markdownコンテンツ (例: RAGやクイズ用)
  /data         # 静的データファイル
  /hooks        # カスタムReactフック
  /pages        # ページコンポーネント (ルーティング)
  /types        # TypeScript型定義
  App.tsx       # メインアプリケーションコンポーネント
  main.tsx      # エントリーポイント
  index.css     # グローバルスタイル
```

## 5. コーディング規約
- **TypeScript:** 強い型付けを使用してください。可能な限り `any` は避けてください。
- **コンポーネント:** フックを使用した関数コンポーネント。小さく、焦点を絞ったものにしてください。
- **ファイル命名:** コンポーネントはPascalCase (`MyComponent.tsx`)、ユーティリティ/フックはcamelCase (`useAuth.ts`)。
- **状態管理:** 必要に応じてReact Contextまたはローカルステートを使用してください。

## 6. ワークフローとベストプラクティス
- **リファクタリング:** 既存のコードを変更する場合は、後方互換性を確保するか、すべての参照を更新してください。
- **テスト:** （該当する場合）重要なロジックのテストを維持してください。
- **ドキュメント:** 主要な機能を追加する際は、`README.md` または `docs/` 内の関連ドキュメントを更新してください。

## 7. 特別な指示
- **RAGボット:** チャットボットのインターフェースは、モダンなチャットUI（ChatGPTなど）を模倣してください。
- **クイズ:** クイズのロジックが堅牢であり、UIが直感的であることを確認してください。
