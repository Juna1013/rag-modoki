# RAGボットガイド

## 🤖 概要

RAG（Retrieval-Augmented Generation）ボットは、Google Gemini APIを活用したAI会話型問題選択システムです。自然な対話を通じて、ユーザーに最適な学習問題を提案します。

## 🎯 RAGとは

**RAG (Retrieval-Augmented Generation)** は、以下のプロセスを組み合わせた手法です：

1. **Retrieval（検索）**: 問題データベースから関連する問題を検索
2. **Augmentation（拡張）**: AIが会話履歴と検索結果を組み合わせる
3. **Generation（生成）**: 文脈に適した自然な応答を生成

## 🏗 システム構成

```
ユーザー入力
    ↓
パターンマッチング（高速応答用）
    ↓
Gemini API（会話理解）
    ↓
問題検索エンジン（キーワードマッチング）
    ↓
難易度フィルタリング
    ↓
結果表示
```

## 💬 使い方

### 基本的な使い方

#### 1. トピックで問題を探す
```
入力: ネットワーク

応答: ネットワークに関連する問題を5問見つけました！✨

見つかった問題：
1. TCPとUDPの違いについて説明している...
2. OSI参照モデルの第4層は何か...
3. BGPプロトコルの役割について...
...

この問題セットでクイズを開始しますか？
```

#### 2. 難易度を指定する
```
入力: 簡単なデータベース

応答: 初級レベルのデータベース問題を探しますね！
```

対応する難易度キーワード：
- **基本**: 簡単、易しい、初級、基本、初心者、基礎
- **中級**: 中級、普通、標準
- **上級**: 上級、難しい、高度、応用、チャレンジ

#### 3. 問題数を指定する
```
入力: セキュリティ10問

応答: セキュリティに関する問題を10問探します！
```

指定できる問題数: **1〜10問**

#### 4. 組み合わせて指定
```
入力: 簡単なネットワーク3問

応答: 初級レベルのネットワーク問題を3問選びますね！
```

## 🔧 技術詳細

### 問題検索アルゴリズム

#### 1. キーワード拡張
ユーザーの入力から関連キーワードを自動的に拡張：

```typescript
const keywordMap = {
  'ネットワーク': ['tcp', 'ip', 'bgp', 'osi', 'プロトコル', ...],
  'データベース': ['rdbms', 'sql', 'acid', '正規化', ...],
  'セキュリティ': ['暗号', 'rsa', '楕円曲線', ...],
  // ...
};
```

#### 2. スコアリング
各問題に対してスコアを計算：
- **完全一致**: +3ポイント
- **部分一致**: +1ポイント

#### 3. ソートとフィルタリング
スコアの高い順にソートし、指定された問題数を返す。

### Gemini API統合

#### プロンプト構造


### フォールバック機能

Gemini APIが利用できない場合、パターンマッチングで代替：

1. **挨拶検出**: `こんにちは`、`おはよう`など
2. **感謝検出**: `ありがとう`、`thanks`など
3. **直接検索**: トピックキーワードを直接検索

## 🐛 トラブルシューティング

### 問題が見つからない
**原因**: キーワードが問題データベースにマッチしない

**解決策**:
- 別のキーワードを試す
- より一般的な用語を使う
- 複数のキーワードを組み合わせる

### AI応答が遅い
**原因**: Gemini APIのレスポンス時間

**解決策**:
- フォールバック機能が自動的に作動
- インターネット接続を確認
- API クォータを確認

### 会話がリセットされる
**原因**: ページリロードで会話履歴が消える

**現状**: 会話履歴はメモリに保存（永続化なし）

**今後の対応**: LocalStorageやデータベースへの保存を検討

## 📈 パフォーマンス

- **初回応答**: 約0.8〜2秒（APIレスポンス時間による）
- **フォールバック**: 即座（パターンマッチング）
- **問題検索**: 約0.1秒以下
- **会話履歴**: 最新4メッセージのみ保持（メモリ最適化）

## 🔐 セキュリティ

- **APIキー**: 環境変数で管理、クライアント側で使用
- **入力検証**: XSS対策として入力をサニタイズ
- **エラーハンドリング**: センシティブな情報を含めない
